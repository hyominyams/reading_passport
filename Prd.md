# 1. 프로젝트 개요

## 한 줄 정의
제3세계 그림책을 읽고, 탐색하고, 자신만의 이야기를 창작하는 글로벌 독서교육 웹앱.

## 핵심 철학
AI가 답을 주는 게 아니라, 학생이 스스로 생각하게 만드는 것.

## 기술 스택
- **프레임워크**: Next.js (App Router)
- **DB/인증**: Supabase
- **배포**: Vercel
- **스타일**: Tailwind CSS + Framer Motion
- **텍스트 AI**: GPT-5-mini (소형모델 병용)
- **이미지 AI**: Nanobanana2 (cref 지원)

## 가입 및 계정 흐름

```
관리자 계정 (시드 데이터로 고정 생성)
    ↓
교사 이메일 가입 → 자동 승인 → 학교/학년/반 설정
    ↓
```

```
  └── 전체 공개 → 관리자 승인 요청 발송
                      ↓
```

# 5. 페이지별 상세 설계

---

## Page 1 — 로그인 (/login)

- 교사 탭: 이메일 + 비밀번호
- 학생 탭: 6자리 코드 입력
- 최초 접속 학생 → 온보딩(닉네임, 아바타) → 국가 선택
- 재접속 학생 → 바로 국가 선택

---

## Page 2 — 국가/책 선택 (/map)

- 세계 지도 or 카드 그리드
- 열려있는 국가에 마커 표시
- 국가 클릭 → 해당 국가 책 목록
- 책 클릭 → 언어 선택 모달 → 표지 & 소개 페이지

---

## Page 3 — 책 표지 & 소개 (/book/[id])

- 그림책 표지 이미지
- 책 제목, 짧은 소개글, 배경 국가 정보
- 현재 내 도장 현황 미리보기 (0/4)
- [탐험 시작하기] → 3분할 카드 선택 화면

---

## Page 4 — 3분할 카드 선택 (/book/[id]/activity)

**인터랙션:**
- 3개 카드 가로 배치
- 기본 상태: 전체 채도 낮음 (흑백/투명)
- 호버 시: 해당 카드만 컬러, 나머지 더 투명
- 완료한 탭: 도장 표시
- My Story: 3개 완료 후 하단에 별도 버튼으로 등장

| 카드 | 이름 | 도장 |
|------|------|------|
| 📖 | Story Read | 완료 시 도장 1 |
| 🌍 | Hidden Stories | 완료 시 도장 2 |
| 💬 | Talk with [캐릭터명] | 완료 시 도장 3 |
| ✏️ | My Story | 완성 시 도장 4 |

## Page 5 — Story Read (/book/[id]/read)

- PDF → 페이지 단위 파싱 → 좌우 슬라이더 (책 넘기듯)
- 구성: 표지 → 그림1 → 텍스트1 → 그림2 → 텍스트2 ...
- 마지막 페이지 도달 시:
  - 감정 스티커 5개 (기쁨/슬픔/용기/놀람/화남)
  - 드래그앤드롭으로 감정 선택
  - 한 줄 감상 텍스트 입력
  - 저장 → 도장 1 획득

---

## Page 6 — Hidden Stories (/book/[id]/explore)

교사/관리자가 책별로 등록한 탐색 콘텐츠 카드 피드.

**콘텐츠 타입별 처리:**

| 타입 | 처리 방식 |
|------|-----------|
| 📹 유튜브 | 앱 안 iframe 재생 |
| 📄 PDF | 앱 안 뷰어 |
| 🖼️ 이미지 | 앱 안 확대 보기 |
| 🔗 외부링크/기사 | 새 탭으로 열기 |

- 일정 수 이상 탐색 완료 시 → 도장 2 획득

---

## Page 7 — Talk with Character (/book/[id]/chat)

### 등장인물 선택 화면
- 책 등록 시 AI 자동 스캔으로 핵심 인물 2~3명 추출
- 인물 카드 가로 배치 (이름 + 역할 + 한 줄 소개)
- 카드 클릭 → 해당 인물과 채팅 시작

### 채팅 화면 구성
```
[인물 아바타 + 이름]        [새 채팅] [☰ 사이드바]
─────────────────────────────────────────────────
채팅 영역                  │ 사이드바 (열고닫기)
                           │ 이전 대화 목록
                           │ - 날짜 + 인물 + 첫 줄
                           │ - 클릭 시 읽기 전용 열람
─────────────────────────────────────────────────
[입력창                              ] [전송]
```

**[새 채팅] 클릭 시:**
- 현재 세션 DB 저장 후 종료
- 인물 선택 화면으로 복귀

### 챗봇 원칙
- 등장인물 1인칭 페르소나 유지
- 감정 전달 중심 대화
- 이야기 바깥 내용 답변 안 함
- 반드시 질문으로 응답 마무리
- 도장 3 획득 조건 없음 (대화 완료 시 자동)

### 부적절 내용 플래그
- 세션 종료 시 소형모델로 자동 스크리닝
- 욕설/폭력/개인정보/자해 관련 → flagged: true
- 교사 대시보드에 ⚠️ 표시

## Page 8 — My Story: 유형 선택 + 게이지 채팅 (/book/[id]/mystory)

### Step 1 — 이야기 유형 선택
```
[이야기 이어쓰기]
[주인공으로 새 이야기 써보기]
[엑스트라 주인공의 뒷이야기 쓰기]
[결말 바꾸기]
[기타: 직접 입력]
```

### Step 2 — 게이지 채팅

**화면 구성:**
- 좌측: 채팅창 (학생이 자유롭게 이야기 재료 입력)
- 우측: 수직 AI 게이지 바 (0~100%)

**게이지 계산 (클라이언트 사이드):**

| 기준 | 점수 |
|------|------|
| 입력 길이 10자 미만 | +3% |
| 입력 길이 10~30자 | +8% |
| 입력 길이 30~60자 | +15% |
| 입력 길이 60자 이상 | +20% |
| 현재 질문 항목 키워드 매칭 | +10% 보너스 |
| 5턴 미만 | 최대 80% 제한 |

**4가지 수집 항목:**
- 등장인물 (character)
- 배경/장소 (setting)
- 사건/갈등 (conflict)
- 결말 방향 (ending)

**알림 및 버튼:**
- 90% 이상 → 보조 알림: "이야기를 제작하면 더 좋은 이야기가 나와요! ✨"
- 5턴 이후 → [제출하기] 버튼 활성화

### Step 3 — 소형모델 최종 검증

[제출하기] 클릭 시 1회 실행.

```
통과 → 초안 생성 프롬프트 실행
미달 → 해당 항목 게이지 -10% 감소
       안내 메시지: "이런 내용이 조금 더 필요해요"
       → 학생이 추가 입력 후 재제출
```

### Step 4 — 이야기 초안 생성

검증 통과 후 실행.
입력값: 원문(책 전체) + 학생 답변 전체 + 선택 유형
출력: [PAGE_BREAK] 태그로 구분된 3~5구역 초안 (의도적으로 부족하게)

## Page 10 — 캐릭터 디자인 (/book/[id]/mystory/characters)

1. AI가 학생 이야기에서 핵심 캐릭터 2명 자동 추출 + 확인
2. 캐릭터별 쉬운 말로 외모 설명 입력
   예: "갈색 피부, 긴 머리, 전통 옷 입은 소녀"
3. GPT-5-mini가 Nanobanana2용 프롬프트로 고도화 (enhancer)
4. 아트 스타일 선택 (색연필 / 수채화 / 판화 / 파스텔)
5. 이미지 생성 → ref1, ref2로 저장

**시스템 프롬프트 고정값:**
- 흰 배경, 캐릭터만
- 전신 또는 반신
- 선택된 아트 스타일

## Page 11 — 장면별 이미지 생성 (/book/[id]/mystory/scenes)

페이지별 반복:
```
학생 텍스트 확인
→ 장면 설명 입력
→ [생성하기]
→ 프롬프트 + ref(캐릭터명 자동감지) + 스타일 → Nanobanana2
→ 결과 확인 or 재생성
→ 다음 장면
```

**캐릭터 감지 로직:**
- 텍스트에 캐릭터 이름 포함 → 해당 ref 자동 첨부
- 두 명 모두 포함 → ref1 + ref2 함께 전송

## Page 12 — 완성 & 서재 등록 (/book/[id]/mystory/finish)

**결합 구조:**
```
표지 이미지 + 표지 텍스트
그림1 + 텍스트1
그림2 + 텍스트2
...
```

**완료 시:**
- 폭죽 애니메이션
- 도장 4 획득
- 여권 해당 국가 페이지 완성
- 서재 자동 등록 (기본: 전체 공개)
- 등록 위치: `국가 책장 → 해당 원작 책 섹션 → 학생 작품`

**서재 공개 설정 (학생 선택):**
- 전체 공개 (기본값)
- 우리 반만
- 나만 보기

**PDF 다운로드:**
- 작성 언어 버전
- 번역본 (영어 ↔ 한국어) GPT-5-mini로 생성 후 다운로드
- 서재에 원본 + 번역본 함께 등록, 언어 토글 버튼 제공

## Page 13 — 서재 (/library)

```
국가/책별 서재
  └── 콜롬비아 - 와유족 이야기
        └── [작품A] [작품B] [작품C] ...
```

### 정보 구조
```
[국가 책장]
  └── [원작 책 섹션]
        ├── 원작 표지 / 제목 / 작품 수
        └── [학생 작품] [학생 작품] [학생 작품] ...
```

- 1단계 계층: 국가별 책장
- 2단계 계층: 원작 책별 섹션 (`book_id` 기준)
- 3단계 계층: 해당 원작에서 나온 학생 작품 컬렉션
- 같은 국가 안에서도 원작 책이 다르면 반드시 섹션을 분리한다

### UX / 디자인 방향
- Book Creator를 그대로 복제하지 않고, `전시형 그림책 서가` 톤으로 구현
- 국가 섹션 상단에는 국가 이미지 스트립, 국기, 작품 수를 배치
- 원작 책 섹션은 원작 표지를 대표 카드처럼 크게 보여준다
- 학생 작품 카드는 첫 장면 이미지를 표지처럼 사용하고, 얇은 책등 그림자와 hover 모션으로 책 느낌을 준다
- 모바일에서는 국가 섹션과 원작 책 섹션을 세로 스택으로 배치한다

### 상호작용
- 클릭 → 그림책 뷰어로 열림
- 뷰어에서 마지막 페이지까지 도달하면 `읽기 완료`로 기록
- 언어 토글 버튼 (원본 / 번역본)
- 좋아요: 로그인 사용자 누구나 가능
- 댓글: `타인 작품을 끝까지 읽은 사용자`만 가능
- 자기 작품에는 댓글을 남길 수 없다
- 댓글은 작품 마지막 페이지 하단에서 작성한다

### 정렬 / 노출 기준
- 기본 정렬은 원작 책 섹션 안에서 좋아요 순
- 추후 옵션: 최신순 / 조회순
- `private` 작품은 서재에 노출하지 않는다
- `class` 공개 작품은 권한이 있는 사용자에게만 노출한다

## Page 14 — 교사 대시보드 (/teacher)

### 1단계 — 반 전체 현황
```
이름      현재 책          도장   최근활동   알림
────────────────────────────────────────────
```

# 11. 도서관 리뉴얼 계획

## 11-1. 목표
- 도서관을 단순 카드 모음이 아니라 `국가 > 원작책 > 학생 작품` 구조의 컬렉션으로 재구성한다
- 학생 작품이 어떤 나라와 어떤 원작 책에서 출발했는지 한눈에 보이게 만든다
- 좋아요와 읽기 완료 기반 댓글을 통해 가벼운 사회적 상호작용을 제공한다

## 11-2. 구현 범위
1. 데이터 조회 구조 개편
   - `library + stories + books + author` 조인
   - `country_id`로 1차 그룹핑, `book_id`로 2차 그룹핑
2. 서가 레이아웃 개편
   - 국가별 책장 섹션
   - 원작 책 대표 카드
   - 학생 작품 카드 그리드
3. 그림책 뷰어 보강
   - 원본/번역 토글 유지
   - 마지막 페이지 도달 시 읽기 완료 기록 저장
4. 사회적 상호작용 규칙 적용
   - 좋아요는 로그인 사용자 누구나 가능
   - 댓글은 `타인 작품`을 `끝까지 읽은 뒤`에만 가능
5. 후속 다듬기
   - 국가 스트립 이미지, 책 hover, empty state, 모바일 아코디언 최적화

## 11-3. 구현 순서
1. PRD 기준으로 `/library` 데이터 모델과 그룹핑 구조를 먼저 변경
